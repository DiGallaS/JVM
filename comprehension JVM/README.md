

## Задание 1. Понимание JVM.:



### 1 шаг.
```
Начинается с подзагрузка класса JvmComprehension через подсистему загрузчиков классов.
Загрузчики передают полученный class друг другу до BootstrapClassLoader. 
После чего начинается поиск и в случае неудачи передача предыдущим classloader's и так пока класс не загрузят.
Следующий этап Linking(связывание): 
Проверяется что код валиден, подготовка примитивов в static полях и связывание ссылок на другие классы.
После Linking начинает инициализация класса,а точнее инициализация статических методов main(String[] args) и printAll().
и все это загружается в область памяти - Metaspace.
```
### 2 шаг.
```
JVM начинает выполнять по строчно загруженный класс. При вызове метода main() создается фрейм в Stack Memory.
*Строка 1. В созданный фрейм записывается примитивная переменная int i = 1.
*Строка 2. В heap загружается объект Object. Затем загружается переменная "о" в фрейме и ей присваивается созданный ранее объект.
*Строка 3. Аналогично строке 2 происходит с Integer ii = 2.
*Строка 4. Создается новый фрейм printAll() c стремя параметрами: int i и две ссылочные переменные Object o и Integer ii. 
Начинается выполнение этого метода.
*Строка 5. Аналогично строке 2 происходит запись в фрейм printAll() Integer uselessVar = 700.
*Строка 6. Создается новый фрейм println(),  который ссылается на переменные "о" ,
она ссылается на метод toString() класса String(по идее тоже должен создаться фрейм), и переменные ii и i. 
Результат "java.lang.Object@2d98a33512" отправляется в консоль. 
После выполнения 6 строки возвращаемся к методу main().
Т.к. все вызовы на методы printAll(), println() и toString()выполнены то их фреймы удаляются из стека.
*Строка 7. Заново создается фрейм println() и результат "finished" отправляется в консоль. 
После чего фрем main() и println() так же удаляется. 
Когда сборщик мусора сработает все ненужные данные будут уничтожены из кучи.
```
